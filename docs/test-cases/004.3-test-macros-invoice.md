# Test Case 004.3: Test Macros - InvoiceForServicesOrSaas

**Feature:** Macro System - InvoiceForServicesOrSaas Macro  
**Date:** 2026-04-06  

## Preconditions

See [004.1-test-macros.md](./004.1-test-macros.md) for general macro testing preconditions.

## Test Objective

Verify that the InvoiceForServicesOrSaas macro correctly creates transactions for invoicing customers for services rendered or SaaS subscriptions.

The test will validate:
- InvoiceForServicesOrSaas macro parameter form with 7 parameters
- Default value resolution for `{next_invoice_SI}` placeholder
- Default value resolution for `{today}` placeholder
- Partner selection for customers
- Account filtering for revenue accounts (3..) and receivable accounts (1100)
- Transaction creation that recognizes revenue and creates a receivable
- Automatic invoice number generation and tagging

## Macro Under Test: InvoiceForServicesOrSaas

## Acceptance Criteria

- [ ] InvoiceForServicesOrSaas macro is available in macro selection
- [ ] Parameter form displays all 7 required fields with correct types
- [ ] Default value `{today}` is displayed as a placeholder in the UI
- [ ] Default value `{next_invoice_SI}` is displayed as a placeholder in the UI
- [ ] Placeholders are resolved during macro execution on the backend
- [ ] Partner dropdown allows selection of customers
- [ ] Invoice number field accepts 8-digit invoice numbers with SI prefix
- [ ] Revenue account dropdown is filtered to pattern ^3.*:3400.*$|^3.*:3600.*$
- [ ] Receivable account dropdown is filtered to pattern ^1.*:10.*:110.*:1100.*$
- [ ] Required field validation works correctly
- [ ] Transaction credits revenue account (increases revenue)
- [ ] Transaction debits receivable account (increases asset - customer owes money)
- [ ] Created transactions are properly balanced
- [ ] Invoice tag is automatically created with format invoice:SI00000001
- [ ] Account balances are updated correctly
- [ ] Transactions appear in transaction list with correct tags
- [ ] Balance sheet equation holds after transactions
- [ ] Sequential invoice numbering works correctly

## Notes

- This macro is used to **invoice customers for services or SaaS subscriptions**
- Revenue is recognized when the invoice is sent (accrual accounting principle)
- The receivable represents money owed by the customer
- Example use case: Invoicing a customer CHF 7 for consulting services
- The transaction increases both revenue (credit) and receivables (debit)
- Invoice numbers follow the format: SI + 8 digits (e.g., SI00000001, SI00000002)
- The macro automatically generates the next available invoice number
- The `{today}` placeholder is resolved to the current date
- The `{next_invoice_SI}` placeholder is resolved to the next invoice number in the sequence
