# Test Case 004.5: Test Macros - CustomerPaysInvoice

**Feature:** Macro System - CustomerPaysInvoice Macro  
**Date:** 2026-04-06  

## Preconditions

See [004.1-test-macros.md](./004.1-test-macros.md) for general macro testing preconditions.

Additionally:
- Invoice SI00000001 must exist (created in test 004.3)
- Invoice SI00000001 must be unpaid (receivable balance exists)
- Partner P00000014 must exist

## Test Objective

Verify that the CustomerPaysInvoice macro correctly creates transactions for recording customer payments against previously issued invoices.

The test will validate:
- CustomerPaysInvoice macro parameter form with 5 parameters
- Invoice autocomplete with regex search support (e.g., `.*01` to find invoices ending in 01)
- Partner selection matching the invoice
- Account filtering for bank accounts and receivable accounts
- Transaction creation that reduces receivables and increases bank balance
- Invoice tag preservation from the original invoice

## Macro Under Test: CustomerPaysInvoice

## Acceptance Criteria

- [ ] CustomerPaysInvoice macro is available in macro selection
- [ ] Parameter form displays all 5 required fields with correct types
- [ ] Invoice autocomplete supports regex search patterns (e.g., `.*01`, `SI.*`)
- [ ] Invoice autocomplete displays matching invoices in descending order (newest first)
- [ ] Partner dropdown allows selection of customers
- [ ] Bank account dropdown is filtered to pattern ^1.*:10.*:100.*:1020.*$
- [ ] Receivable account dropdown is filtered to pattern ^1.*:10.*:110.*:1100.*$
- [ ] Required field validation works correctly
- [ ] Transaction debits bank account (increases bank balance)
- [ ] Transaction credits receivable account (decreases receivable - customer paid)
- [ ] Created transactions are properly balanced
- [ ] Invoice tag is preserved from the original invoice
- [ ] Account balances are updated correctly
- [ ] Transactions appear in transaction list with correct tags
- [ ] Balance sheet equation holds after transactions
- [ ] Multiple transactions can reference the same invoice tag

## Notes

- This macro is used to **record customer payments against previously issued invoices**
- The payment reduces the receivable (customer no longer owes money)
- The payment increases the bank balance (money received)
- Example use case: Customer P00000014 pays invoice SI00000001 for CHF 7.00
- The transaction decreases receivables (credit) and increases bank (debit)
- Invoice tags link the payment to the original invoice
- Regex search allows flexible invoice lookup (e.g., `.*01` finds all invoices ending in 01)
- The macro automatically uses the same invoice tag as the original invoice
- This creates a complete audit trail for each invoice (issuance + payment)
