# Test Case 004.2: Test Macros - RepayStaff

**Feature:** Macro System - RepayStaff Macro  
**Date:** 2026-04-06  

## Preconditions

See [004.1-test-macros.md](./004.1-test-macros.md) for general macro testing preconditions.

## Test Objective

Verify that the RepayStaff macro correctly creates transactions to repay money owed to staff members for expenses they paid on behalf of the company.

The test will validate:
- RepayStaff macro parameter form with 7 parameters
- Partner selection for staff members
- Invoice number list validation (comma-separated, 8 digits each)
- Account filtering for bank accounts (1..) and staff liability accounts (2..)
- Transaction creation that clears staff liability

## Macro Under Test: RepayStaff

### Scenarios: Execute RepayStaff macro to repay staff member

```gherkin
Feature: Macro System - RepayStaff

  Background:
    Given the user is signed into the application
    And the journal "Abstratium 2026" exists with a complete account tree
    And opening balances have been established for 2026-01-01
    And initial business transactions have been recorded
    And the user is on the journal detail page for "Abstratium 2026"

  Scenario: Repay staff member using RepayStaff macro
    When the user navigates to the "Transactions" section
    And the user clicks "Create Transaction from Macro" or "Use Macro"
    Then the macro selection interface should be displayed
    And the RepayStaff macro should be available in the macro list
    
    When the user selects the "RepayStaff" macro from the dropdown
    Then the macro parameter form should be displayed
    And all 7 parameters should be displayed with appropriate UI controls
    And the form should show the following fields:
      | Field | Type | Default Value | Required |
      | Payment date | Date picker | {today} | Yes |
      | Partner (staff member) | Partner dropdown | (empty) | Yes |
      | Description | Text input | (empty) | Yes |
      | Invoice numbers to reimburse | Text input | (empty) | Yes |
      | Total amount | Number input | (empty) | Yes |
      | Bank account | Account dropdown | (empty) | Yes |
      | Staff member account | Account dropdown | (empty) | Yes |
    And default values should be pre-filled where specified
    And required fields should be clearly marked
    
    When the user enters "2026-08-02" as the payment date
    And the user selects partner "P00000001" as the staff member
    And the user enters "Test macros 004.2" as the description
    And the user enters "PI00000002,PI00000003" as the invoice numbers
    And the user enters "38.50" as the total amount
    And the user selects account "1020 Bank Account (asset)" as the bank account
    And the user selects account "2210.001" as the staff member account
    Then all required fields should be filled
    And the form should pass validation
    
    When the user clicks "Execute Macro"
    Then the transaction should be created successfully via the macro system
    And the transaction should be persisted to the database
    And the transaction should be balanced (debits = credits)
    And the journal should be displayed

    When the user views the newly created transaction
    Then the transaction should have the following properties:
      | Property | Value |
      | Date | 2026-08-02 |
      | Partner | P00000001 |
      | Description | Test macros 004.2 |
      | Status | Posted (*) |
      | Tags | :Payment: |
    And the transaction should have 2 entries
    And entry 1 should credit account "1020 Bank Account (asset)" for CHF -38.50
    And entry 2 should debit account "2210.001" for CHF 38.50
    And the tag `:Payment:` should be automatically added
    And the invoice numbers "PI00000002,PI00000003" should be stored in the transaction metadata
    And the transaction should have a unique ID
    And the transaction should be linked to the correct journal
    
    When the user views account balances as of "2026-08-02"
    Then the "1020 Bank Account (asset)" account balance should have decreased by CHF 38.50
    And the "2210.001" staff liability account balance should have decreased by CHF 38.50
    And the staff liability is reduced (debit reduces liability)
    And the bank account is reduced (credit reduces asset)
    And all other account balances should remain unchanged
    And the balance sheet equation should hold

  Scenario: Validate macro parameter filtering for RepayStaff
    When the user selects the "RepayStaff" macro
    And the user clicks on the "Bank account" dropdown
    Then only accounts matching the pattern "^1.*:10.*:100.*:1020.*$" should be displayed
    And account "1020 Bank Account (asset)" should be available
    And account "1000 Cash" should NOT be available
    
    When the user clicks on the "Staff member account" dropdown
    Then only accounts matching the pattern "^2.*:20.*:220.*:2210.*:2210.*$" should be displayed
    And account "2210.001" should be available
    And account "2000 Liabilities" should NOT be available

  Scenario: Validate invoice number format
    When the user selects the "RepayStaff" macro
    And the user enters "PI0002" as the invoice numbers (too short)
    And the user attempts to create the transaction
    Then a validation error should be displayed
    And the error should indicate proper format: "comma-separated, 8 digits each"
    
    When the user enters "PI00000002,PI00000003" as the invoice numbers (correct format)
    Then the validation should pass
    And the invoice numbers should be accepted

  Scenario: Verify transaction appears in transaction list
    Given the user has created a transaction using the RepayStaff macro
    When the user views the transaction list for "Abstratium 2026"
    And the user filters by date "2026-08-02"
    Then the transaction "Test macros 004.2" should appear in the list
    
    When the user filters by partner "P00000001"
    Then the transaction "Test macros 004.2" should appear in the list
    
    When the user filters by tag ":Payment:"
    Then the transaction "Test macros 004.2" should appear in the list
```

## Acceptance Criteria

- [ ] RepayStaff macro is available in macro selection
- [ ] Parameter form displays all 7 required fields with correct types
- [ ] Partner dropdown allows selection of staff members
- [ ] Invoice numbers field accepts comma-separated 8-digit values
- [ ] Bank account dropdown is filtered to pattern ^1.*:10.*:100.*:1020.*$
- [ ] Staff account dropdown is filtered to pattern ^2.*:20.*:220.*:2210.*:2210.*$
- [ ] Required field validation works correctly
- [ ] Transaction debits staff liability account (reduces what company owes)
- [ ] Transaction credits bank account (cash goes out)
- [ ] Created transactions are properly balanced
- [ ] Account balances are updated correctly
- [ ] Transactions appear in transaction list with correct tags
- [ ] Balance sheet equation holds after transactions

## Notes

- This macro is used to **repay money owed to staff members** for expenses they paid on behalf of the company
- It clears the liability created by the PaymentByStaff macro
- Example use case: Repaying a staff member for domain names they purchased for the company
- The transaction reduces both the staff liability (debit) and the bank account (credit)
- Invoice numbers should reference the original invoices being reimbursed
